# Intake local — Zalmar Travel

Este directorio es el área privada de intake para materiales fuente de Zalmar Travel. No todo lo que vive aquí debe publicarse: los archivos se revisan, se documentan y solo se promueven a `public/` o `src/data` cuando hay una intención explícita de uso público.

## Criterio de este pase

- Las capturas nuevas en `screenshots/` se documentan como pantallas del MVP en español.
- El uso de español es una decisión práctica de desarrollo ágil: es el idioma principal de Caleb y permite iterar producto, copy, criterios funcionales y validaciones técnicas con menos fricción.
- El PDF maestro funcional del paquete de diseñador queda como fuente interna de análisis y referencia, no como documento adjunto ni enlace público del portfolio.
- Los nuevos assets de branding y el pitch deck quedan registrados como intake; su promoción pública requiere una revisión posterior.

## Estructura

| Ruta | Rol | Estado público |
|---|---|---|
| `branding/` | Logos/manual assets agregados manualmente para revisar identidad visual actual de Zalmar. | Intake privado; no promovido automáticamente. |
| `pitch-deck/` | Pitch deck de Zalmar para contexto narrativo, producto y presentación. | Intake privado; no enlazado públicamente en este pase. |
| `screenshots/` | Capturas del MVP en español y referencias visuales del estado actual del producto. | Algunas capturas equivalentes ya están promovidas como imágenes públicas seleccionadas; mantener revisión por archivo. |
| `zalmar-travel-paquete-designer-pdf-y-branding-v1.1/` | Paquete fuente para diseñador con documentos, branding base, Zaly y referencias visuales. | Fuente interna; promover solo outputs públicos revisados. |

## Notas de publicación

- El documento `01-documento-maestro-funcional-frontend-zalmar-travel-v1.1.pdf` se marca como `internal-reference-only`.
- Si se necesita evidenciar el proyecto públicamente, preferir capturas seleccionadas, captions pulidas y explicación del MVP antes que exponer documentos funcionales completos.
- Los nombres normalizados en este directorio usan minúsculas, guiones y sin acentos; los nombres originales se conservan en `manifest.md` cuando aportan trazabilidad.
